Notes de session Homelab : Configuration initiale et réseau de Proxmox VE

Aperçu de la session

Date : 22 octobre 2025 Objectif : Installation de Proxmox VE, configuration réseau et préparation initiale de la VM Matériel : Lenovo ThinkCentre M920q

Spécifications Matérielles

Détails du serveur

Modèle : Lenovo ThinkCentre M920q (Format Ultra Compact)

Spécifications : - CPU : Intel i5-8500T (6 cœurs, cache 9 Mo, TDP 35W) - RAM : 16 Go DDR4 - Stockage Principal : SSD 512 Go (NVMe ou SATA) - Stockage Secondaire : HDD 250 Go (pour les sauvegardes/images ISO) - Réseau : Gigabit Ethernet (enp3s0) - Format : 1.0L ultra-compact

Pourquoi ce matériel : - Faible consommation d'énergie (TDP de 35 W idéal pour un fonctionnement 24/7) - Format compact (parfait pour un homelab) - RAM suffisante pour plusieurs VM - Double stockage pour la séparation système/données - Excellent rapport prix/performance sur le marché de l'occasion

Configuration Réseau

IP de l'hôte Proxmox : 192.168.11.254 Passerelle/Routeur : 192.168.11.1 Masque de sous-réseau : 255.255.255.0 (/24) Réseau : 192.168.11.0/24

Étape 1 : Accès à l'interface web de Proxmox

Accès initial

# From web browser on local network
https://192.168.11.254:8006

# Login credentials
Username: root
Domain: pam (Linux PAM authentication)
Password: [set during installation]

Liste de vérification après la première connexion

Vérifier : - [ ] L'interface web se charge (ignorer l'avertissement SSL pour le certificat auto-signé) - [ ] La connexion avec les identifiants root est possible - [ ] Le tableau de bord affiche correctement les ressources du serveur - [ ] L'interface réseau affiche la bonne adresse IP

Configuration après la connexion

Désactiver le dépôt Enterprise (si vous utilisez la version gratuite) :

# SSH into Proxmox
ssh root@192.168.11.254

# Edit sources list
nano /etc/apt/sources.list.d/pve-enterprise.list

# Comment out enterprise repo
# deb https://enterprise.proxmox.com/debian/pve bullseye pve-enterprise

# Add no-subscription repository
echo "deb http://download.proxmox.com/debian/pve bullseye pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list

# Update
apt update

Étape 2 : Connexion Ethernet Directe

Configuration physique

Topologie :

Laptop (192.168.11.1)
    ↓ Ethernet Cable
Proxmox Server (192.168.11.254)

Configuration IP Manuelle

Sur l'ordinateur portable (en supposant Linux) :

# Assign static IP to ethernet interface
sudo ip addr add 192.168.11.1/24 dev eth0
sudo ip link set eth0 up

# Test connectivity
ping 192.168.11.254

# Expected: replies from 192.168.11.254

Objectif : - Connexion de gestion directe - Aucune dépendance vis-à-vis du routeur/DHCP - Utile pour le dépannage des problèmes réseau - Simule un réseau de laboratoire isolé

Étape 3 : Vérification du réseau sans fil

Vérifier les interfaces disponibles

# List all network interfaces
ip a

# Expected output includes:
# lo: loopback
# enp3s0: ethernet (primary)
# wlp0s20f3: wireless (if present)

Détecter le matériel réseau

# List PCI network devices
lspci | grep -i network

# Expected output:
# 00:1f.6 Ethernet controller: Intel Corporation...
# (May show wireless if available)

# Check wireless interfaces
iw dev

# List USB devices (for USB WiFi adapters)
lsusb | grep -i wireless

Configuration WiFi (si applicable)

# Install wireless tools
apt install wpasupplicant wireless-tools -y

# Scan for networks
iwlist scan

# Configure wpa_supplicant
wpa_passphrase "SSID" "password" >> /etc/wpa_supplicant/wpa_supplicant.conf

# Connect
wpa_supplicant -B -i wlp0s20f3 -c /etc/wpa_supplicant/wpa_supplicant.conf
dhclient wlp0s20f3

Note : L'utilisation du WiFi sur l'hôte Proxmox n'est généralement pas recommandée pour le pontage réseau des VM.

Étape 4 : Modification de l'adresse IP de Proxmox

Modifier la configuration réseau

# Edit network interfaces file
nano /etc/network/interfaces

Exemple de configuration :

# Loopback interface
auto lo
iface lo inet loopback

# Primary network interface
auto enp3s0
iface enp3s0 inet manual

# Bridge for VMs (management network)
auto vmbr0
iface vmbr0 inet static
    address 192.168.11.254/24
    gateway 192.168.11.1
    bridge-ports enp3s0
    bridge-stp off
    bridge-fd 0
    bridge-vlan-aware yes
    # DNS servers
    dns-nameservers 8.8.8.8 8.8.4.4

Appliquer les modifications réseau

# Method 1: Restart networking service
systemctl restart networking

# Method 2: Reload interfaces (no disruption to VMs)
ifreload -a

# Method 3: Full reboot (safest)
reboot

Vérifier la nouvelle configuration

# Check IP address
ip addr show vmbr0

# Test gateway
ping 192.168.11.1

# Test internet
ping 8.8.8.8
ping google.com

Étape 5 : Téléchargement des images ISO

Méthode 1 : Téléchargement via l'interface web

Via l'interface utilisateur Proxmox :

  1. Naviguer vers : Datacenter → Node → local (storage) → Content
  2. Cliquer sur Upload (Télécharger)
  3. Sélectionner ISO image (Image ISO)
  4. Choisir le fichier depuis l'ordinateur local
  5. Cliquer sur Upload (Télécharger)

Avantages : - Convivial - Pas besoin de ligne de commande - Barre de progression visuelle

Inconvénients : - Lent pour les fichiers volumineux - Nécessite une connexion réseau stable - Dépend du navigateur

Méthode 2 : SCP (Secure Copy)

Depuis l'ordinateur portable/poste de travail :

# Upload single ISO
scp /path/to/file.iso root@192.168.11.254:/var/lib/vz/template/iso/

# Upload multiple ISOs
scp *.iso root@192.168.11.254:/var/lib/vz/template/iso/

# With progress meter
rsync --progress -avz file.iso root@192.168.11.254:/var/lib/vz/template/iso/

Avantages : - Plus rapide pour les fichiers volumineux - Peut reprendre les transferts interrompus (avec rsync) - Scriptable

Inconvénients : - Nécessite un accès SSH - Nécessite des connaissances en ligne de commande

Méthode 3 : Téléchargement direct sur Proxmox

# SSH into Proxmox
ssh root@192.168.11.254

# Navigate to ISO directory
cd /var/lib/vz/template/iso/

# Download directly
wget https://releases.ubuntu.com/22.04/ubuntu-22.04.3-live-server-amd64.iso

# Or with aria2 (faster, multi-connection)
aria2c -x 16 https://releases.ubuntu.com/22.04/ubuntu-22.04.3-live-server-amd64.iso

Avantages : - Méthode la plus rapide - Pas de surcoût de transfert - Directement depuis la source

Inconvénients : - Nécessite Internet sur l'hôte Proxmox - Utilise la bande passante de Proxmox

Étape 6 : Sélection du modèle LLM

Modèles recommandés pour 16 Go de RAM

Niveau 1 : Léger (2-4 Go de RAM)

Mistral 7B Q4 : - Taille : ~4,3 Go - Performance : Excellent usage général - Cas d'utilisation : Code, chat, requêtes générales

Phi-2 (2.7B) : - Taille : ~1,6 Go - Performance : Rapide, bon pour le codage - Cas d'utilisation : Complétion de code, requêtes rapides

Gemma 2B : - Taille : ~1,4 Go - Performance : Très rapide - Cas d'utilisation : Tâches légères, tests

Outils compatibles

Ollama :

# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull model
ollama pull mistral:7b

# Run model
ollama run mistral:7b

llama.cpp :

# Build from source
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
make

# Run model
./main -m models/mistral-7b-q4.gguf -p "Your prompt here"

MCP via Docker :

version: '3.8'
services:
  mcp:
    image: modularcopilot/mcp:latest
    ports:
      - "5000:5000"
    volumes:
      - ./models:/models
    environment:
      - MODEL_PATH=/models/mistral-7b-q4.gguf

Étape 7 : Installation de Docker + MCP + LLM

Installer Docker

# Update system
apt update && apt upgrade -y

# Install dependencies
apt install -y apt-transport-https ca-certificates curl gnupg lsb-release

# Add Docker GPG key
curl -fsSL https://download.docker.com/linux/debian/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Add Docker repository
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker
apt update
apt install -y docker-ce docker-ce-cli containerd.io

# Verify installation
docker --version
docker run hello-world

Déployer MCP

# Pull MCP image
docker pull ghcr.io/modularcopilot/mcp:latest

# Run MCP container
docker run -d \
  --name mcp \
  -p 5000:5000 \
  -v $(pwd)/mcp_data:/app/data \
  ghcr.io/modularcopilot/mcp:latest

# Check logs
docker logs -f mcp

Installer Ollama pour la gestion des LLM

# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull models
ollama pull mistral:7b
ollama pull phi

# List installed models
ollama list

# Run model
ollama run mistral:7b

Résumé de la session

Tâches Complétées : - ✅ Accès à l'interface web Proxmox configuré - ✅ Connexion Ethernet directe établie - ✅ Interfaces réseau vérifiées - ✅ Adresse IP de Proxmox modifiée - ✅ Méthodes de téléchargement ISO documentées - ✅ Modèles LLM identifiés pour le matériel - ✅ Docker + MCP installation planifiée - ✅ Solutions de prise de notes évaluées

Prochaines étapes : 1. Créer la première VM (pfSense ou Ubuntu) 2. Configurer les ponts réseau (vmbr1 pour l'interne) 3. Déployer la pile LLM (Ollama + modèle choisi) 4. Configurer Obsidian avec synchronisation auto-hébergée 5. Créer des snapshots de VM pour la sauvegarde

Notes et Bonnes Pratiques

Configuration Réseau

  • Toujours utiliser le pontage réseau (vmbr0) pour la gestion des VM
  • Créer un pont séparé (vmbr1) pour les réseaux VM isolés
  • Documenter les allocations IP dans le carnet de notes homelab
  • Tester la connectivité de la passerelle après les changements d'IP

Gestion du Stockage

  • Utiliser le SSD pour le système Proxmox et les disques VM
  • Utiliser le HDD pour les sauvegardes et le stockage ISO
  • Activer le thin provisioning pour une utilisation efficace de l'espace
  • Faire des snapshots réguliers avant les changements majeurs

Sécurité

  • Changer immédiatement les mots de passe par défaut
  • Désactiver le dépôt Enterprise pour éviter les erreurs de mise à jour
  • Maintenir Proxmox à jour (apt update && apt upgrade)
  • Utiliser des clés SSH au lieu de mots de passe lorsque cela est possible
  • Envisager la 2FA pour l'interface web Proxmox

Conclusion

La configuration initiale de Proxmox est terminée avec une configuration réseau fonctionnelle, prête pour le déploiement des VM. Le matériel s'avère capable de prendre en charge les services homelab prévus, y compris le déploiement LLM et les applications conteneurisées.

Enseignement clé : Le PC de petit format (ThinkCentre M920q) offre une excellente base de homelab avec une faible consommation d'énergie et des performances suffisantes pour plusieurs VM.

Durée de la session : ~2 heures, y compris la documentation
