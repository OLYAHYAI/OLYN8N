Difficultés rencontrées : Extension du Disque de Ma VM Ubuntu sur Proxmox

Le Problème

Vous avez augmenté la taille du disque dans Proxmox (de 32 Go à 100 Go), mais Ubuntu n'affiche toujours que 32 Go disponibles lorsque vous exécutez df -h. Le disque a été redimensionné au niveau de l'hyperviseur, mais le système d'exploitation n'a pas encore reconnu le nouvel espace.

Ceci est un scénario courant dans la gestion de homelab, et comprendre la résolution appropriée démontre une bonne maîtrise de l'infrastructure.

Comprendre les Couches

Lorsque vous redimensionnez des disques de VM, vous travaillez avec trois couches distinctes :

Couche 1 : Hyperviseur (Proxmox)
   ↓ Le redimensionnement du disque se fait ici (32 Go → 100 Go)
Couche 2 : Table de Partition
   ↓ La partition doit être étendue
Couche 3 : Système de Fichiers
   ↓ Le système de fichiers doit être étendu pour remplir la partition

Chaque couche doit être mise à jour séparément.

Situation Initiale

Avant le Redimensionnement :

  • Proxmox affiche : Disque de 32 Go
  • Ubuntu affiche : 32 Go disponibles sur /

Après le Redimensionnement dans Proxmox :

  • Proxmox affiche : Disque de 100 Go ✓
  • Ubuntu affiche : 32 Go disponibles sur / ❌

Écart : 68 Go d'espace non alloué

Résolution Étape par Étape

Étape 1 : Vérifier la Nouvelle Taille du Disque

# Vérifier le disque et la disposition des partitions
lsblk

# Exemple de sortie :
NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda      8:0    0  100G  0 disk           # ✓ Le disque affiche 100 Go
├─sda1   8:1    0    1M  0 part           # EFI/BIOS boot
└─sda2   8:2    0   32G  0 part /         # ❌ La partition est toujours à 32 Go

Observation Clé : - sda (disque) affiche 100 Go ✓ - sda2 (partition) affiche 32 Go ❌ - Espace non alloué : 68 Go (100 Go - 32 Go)

Étape 2 : Installer les Outils Nécessaires

# Installer cloud-guest-utils (fournit growpart)
sudo apt update
sudo apt install cloud-guest-utils -y

# Alternative : Installer parted (si growpart n'est pas disponible)
sudo apt install parted -y

Étape 3 : Étendre la Partition

Utilisation de growpart (Recommandé) :

# Étendre la partition 2 sur /dev/sda pour remplir l'espace disponible
sudo growpart /dev/sda 2

# Sortie attendue :
# CHANGED: partition=2 start=2048 old: size=67106816 end=67108864 new: size=209713119 end=209715167

Utilisation de parted (Alternative) :

# Lancer parted
sudo parted /dev/sda

# Dans parted :
(parted) print                    # Afficher la disposition actuelle
(parted) resizepart 2 100%        # Redimensionner la partition 2 à 100 % du disque
(parted) quit                     # Quitter parted

# Vérifier
lsblk

Étape 4 : Étendre le Système de Fichiers

La partition est maintenant plus grande, mais le système de fichiers croit toujours qu'il est de 32 Go. Nous devons étendre le système de fichiers pour qu'il remplisse la partition.

Pour les Systèmes de Fichiers ext4 (Les plus courants) :

# Redimensionner le système de fichiers ext4 sur la partition 2
sudo resize2fs /dev/sda2

# Sortie attendue :
# resize2fs 1.46.5 (30-Dec-2021)
# Filesystem at /dev/sda2 is mounted on /; on-line resizing required
# old_desc_blocks = 4, new_desc_blocks = 13
# The filesystem on /dev/sda2 is now 26214139 (4k) blocks long.

Pour les Systèmes de Fichiers XFS :

# Redimensionner le système de fichiers XFS monté sur /
sudo xfs_growfs /

# Ou spécifier le périphérique
sudo xfs_growfs /dev/sda2

Pour les Volumes Logiques LVM :

Si vous utilisez LVM (vérifiez avec lsblk) :

# Étendre le volume physique
sudo pvresize /dev/sda2

# Étendre le volume logique (remplacez vg_name/lv_name par vos noms réels)
sudo lvextend -l +100%FREE /dev/vg_name/lv_name

# Redimensionner le système de fichiers
sudo resize2fs /dev/vg_name/lv_name  # pour ext4
# ou
sudo xfs_growfs /  # pour XFS

Étape 5 : Vérifier les Changements

# Vérifier l'utilisation du disque
df -h

# Sortie attendue :
# Filesystem      Size  Used Avail Use% Mounted on
# /dev/sda2        98G   15G   79G  16% /          # ✓ Affiche maintenant ~100 Go !

# Vérifier avec lsblk
lsblk

# Devrait afficher :
# sda      8:0    0  100G  0 disk
# ├─sda1   8:1    0    1M  0 part
# └─sda2   8:2    0  100G  0 part /         # ✓ La partition est maintenant de 100 Go
