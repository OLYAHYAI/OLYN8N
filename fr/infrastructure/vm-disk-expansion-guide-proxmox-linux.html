Guide d'expansion de disque de VM (Proxmox + Linux)

Tags: #proxmox #storage #filesystem #resize

Référence Rapide

L'expansion d'un disque de VM est un processus en 3 étapes : 1. Proxmox : Augmenter la taille du disque virtuel 2. OS invité : Étendre la partition 3. OS invité : Redimensionner le système de fichiers

Étape 1 : Augmenter la taille du disque dans Proxmox

Via l'interface web

1. Sélectionner la VM → Hardware (Matériel)
2. Cliquer sur Disque Dur (scsi0)
3. Cliquer sur le bouton "Resize disk"
4. Entrer la taille à ajouter (ex. : +50)
5. Cliquer sur "Resize disk"

Via la CLI (Hôte Proxmox)

# Increase VM 100's disk by 50GB
qm resize 100 scsi0 +50G

# Verify
qm config 100 | grep scsi0

Résultat : Le disque virtuel est plus grand, mais l'OS invité ne le voit pas encore.

Étape 2 : Étendre la Partition (À l'intérieur de la VM)

Vérifier la situation actuelle

# View disk and partition layout
lsblk

# Example output:
# sda      8:0    0   150G  0 disk       ← Le disque est de 150G
# └─sda2   8:2    0   100G  0 part /     ← La partition est toujours de 100G

Étendre la Partition

Méthode A : Utilisation de growpart (Recommandé)

# Install if needed
sudo apt install cloud-guest-utils

# Expand partition 2 on /dev/sda
sudo growpart /dev/sda 2

# Verify
lsblk
# sda2 devrait maintenant indiquer 150G

Méthode B : Utilisation de parted

# Interactive mode
sudo parted /dev/sda
(parted) print free          # Show layout
(parted) resizepart 2 100%   # Resize partition 2 to 100%
(parted) quit

# Non-interactive
sudo parted /dev/sda resizepart 2 100%

Méthode C : Utilisation de fdisk (Avancé)

# Only if above methods fail
sudo fdisk /dev/sda
# Type: d, 2 (delete partition 2)
# Type: n, p, 2, <enter>, <enter> (recreate larger)
# Type: w (write changes)
# Reboot if partition in use

Étape 3 : Redimensionner le Système de Fichiers (À l'intérieur de la VM)

Pour le système de fichiers ext4 (Ubuntu/Debian)

# Resize filesystem to fill partition
sudo resize2fs /dev/sda2

# Verify
df -h /

Pour le système de fichiers XFS (CentOS/RHEL)

# Resize XFS filesystem
sudo xfs_growfs /

# Verify
df -h /

Pour le système de fichiers Btrfs

# Resize Btrfs filesystem
sudo btrfs filesystem resize max /

# Verify
df -h /

Exemple Complet (Démarche détaillée)

Scénario : Étendre une VM Ubuntu de 100 Go à 150 Go

Étape 1 : Hôte Proxmox

# Add 50GB to VM 100
qm resize 100 scsi0 +50G

Étape 2 : À l'intérieur de la VM Ubuntu

# Check current state
lsblk
df -h /

# Output shows:
# Disk: 150G, Partition: 100G, Filesystem: 99G

# Expand partition
sudo growpart /dev/sda 2

# Check partition expanded
lsblk
# Output now shows: Partition: 150G

# Expand filesystem
sudo resize2fs /dev/sda2

# Verify final result
df -h /
# Output now shows: Filesystem: 148G (150GB - overhead)

Succès ! Votre VM dispose maintenant de 50 Go d'espace supplémentaire.

Dépannage

Problème : La partition ne s'étend pas

Symptôme :

sudo growpart /dev/sda 2
# Error: no space to grow

Solution :

# Check if Proxmox resize worked
lsblk
# Look at sda size - should be larger

# If sda size is still old:
# - Shutdown VM
# - Resize again in Proxmox
# - Start VM

Problème : Avertissement “GPT PMBR size mismatch”

Symptôme :

sudo fdisk -l /dev/sda
# GPT PMBR size mismatch (209715199 != 314572799)

Solution : Ceci est normal et sera corrigé automatiquement lorsque vous redimensionnerez la partition.

# Fix GPT table
sudo parted /dev/sda
(parted) print
# Answer 'Fix' when prompted
(parted) quit

Problème : “Le système de fichiers est monté, impossible de redimensionner”

Pour ext4 :

# ext4 supports online resizing
sudo resize2fs /dev/sda2
# This works while mounted

Pour XFS :

# XFS must be mounted to resize
sudo xfs_growfs /
# Uses mount point, not device

Pour les autres systèmes :

# May need to unmount or use live CD
# Boot from live USB
# Run resize2fs /dev/sda2
# Reboot

Problème : La partition n'utilise pas la totalité du disque

Vérifier l'espace non alloué :

sudo parted /dev/sda print free

# Shows something like:
# Number  Start   End     Size    File system  Name
#  1      1049kB  2097kB  1049kB               
#  2      2097kB  107GB   107GB   ext4
#         107GB   161GB   53.7GB  Free Space   ← Non alloué

Solution : Étendre la partition (Étape 2 ci-dessus)

Différents schémas de nommage de disques

Type de Disque VM Périphérique Linux Utilisation
IDE /dev/hda Ancien, lent
SCSI /dev/sda Commun
VirtIO Block /dev/vda Le plus rapide (recommandé)
VirtIO SCSI /dev/sda Rapide + support TRIM

Ajuster les commandes en conséquence : - VirtIO : Utiliser /dev/vda au lieu de /dev/sda - Disques multiples : /dev/sdb, /dev/sdc, etc.

Liste de vérification pré-expansion

Avant d'étendre le disque :

  • # In Proxmox
qm config <vmid> | grep scsi0

# In VM
df -h /
lsblk

Vérification post-expansion

# In VM - verify everything
lsblk                    # Disk and partition sizes
df -h                    # Filesystem size and usage
sudo parted /dev/sda print  # Partition table

# Should all show new larger sizes

Tailles de disques courantes

Originale Ajouter Finale Cas d'utilisation
32G +18G 50G Serveur minimal
100G +50G 150G VM de développement
100G +100G 200G Charge de travail de production
200G +300G 500G Base de données/Stockage

Conseil : Ajoutez de l'espace par incréments. Il est plus facile d'ajouter plus tard que de retirer.

Scénario LVM (Processus Différent)

Si votre VM utilise LVM :

# Check if LVM
lsblk
# Look for "lvm" in TYPE column

# Example with LVM
lsblk
# sda
# └─sda2              (physical partition)
#   └─ubuntu--vg-root (LVM volume)

# Resize process:
# 1. Proxmox: Increase disk size
qm resize 100 scsi0 +50G

# 2. VM: Expand partition
sudo growpart /dev/sda 2

# 3. VM: Resize physical volume
sudo pvresize /dev/sda2

# 4. VM: Expand logical volume
sudo lvextend -l +100%FREE /dev/ubuntu-vg/root

# 5. VM: Resize filesystem
sudo resize2fs /dev/ubuntu-vg/root

# Verify
df -h /

Référence rapide des commandes

# View disk layout
lsblk
lsblk -f                        # With filesystem info
sudo parted /dev/sda print      # Detailed partition info
sudo fdisk -l /dev/sda          # Alternative view

# Check filesystem
df -h                           # Mounted filesystems
df -h /                         # Root filesystem specifically
sudo tune2fs -l /dev/sda2       # ext4 details

# Expand operations
sudo growpart /dev/sda 2        # Expand partition
sudo resize2fs /dev/sda2        # Expand ext4
sudo xfs_growfs /               # Expand XFS
sudo btrfs filesystem resize max /  # Expand Btrfs

# Verify
lsblk && df -h /                # Quick check both

Exemple concret

État Initial :

# Proxmox: VM disk = 100GB
# VM: Partition = 100GB  
# VM: Filesystem = 99GB (22GB used)

Commandes Utilisées :

# 1. Proxmox (via web UI):
#    Hardware → Disk → Resize → +50

# 2. Ubuntu VM:
sudo apt install cloud-guest-utils
sudo growpart /dev/sda 2
sudo resize2fs /dev/sda2

État Final :

# Proxmox: VM disk = 150GB
# VM: Partition = 150GB
# VM: Filesystem = 148GB (22GB used, 120GB free)

Résultat : +49 Go d'espace utilisable gagnés ! ✅
