Débloquer l'accès à distance Cloudflare Tunnel Proxmox

2025-11-19

Débloquer l'accès à distance : Mon parcours avec Cloudflare Tunnel et Proxmox

Objectif du projet

Établir un accès à distance sécurisé à l'interface web de Proxmox VE via un sous-domaine public (proxmox.olyhome.site) en utilisant Cloudflare Tunnel, éliminant ainsi le besoin d'exposer des ports directement sur Internet.

Pourquoi Cloudflare Tunnel ?

Problèmes d'accès à distance traditionnels

Approche par redirection de ports : - ❌ Expose les services directement à Internet - ❌ Nécessite une adresse IP statique ou un DNS dynamique - ❌ Vulnérable aux attaques par balayage de ports - ❌ Gestion complexe du pare-feu

Avantages de Cloudflare Tunnel : - ✅ Aucun port exposé sur le pare-feu - ✅ Protection DDoS intégrée - ✅ Certificats SSL gratuits - ✅ Fonctionne avec les adresses IP dynamiques - ✅ Contrôle d'accès via Cloudflare Zero Trust

Aperçu de l'infrastructure

Internet Users
    ↓
Cloudflare Edge Network
    ↓
Cloudflare Tunnel (cloudflared)
    ↓
Proxmox Host (192.168.11.254:8006)

Processus d'installation

Étape 1 : Prérequis

Exigences : - Nom de domaine (par exemple, olyhome.site) - Compte Cloudflare (le niveau gratuit est suffisant) - Proxmox VE installé et accessible sur le réseau local - Système Linux pour exécuter cloudflared

Étape 2 : Configurer le domaine dans Cloudflare

  1. Ajouter un site à Cloudflare :
    • Connectez-vous au tableau de bord Cloudflare
    • Cliquez sur « Ajouter un site »
    • Entrez le domaine : olyhome.site
    • Choisissez le plan Gratuit
  2. Mettre à jour les serveurs de noms :
    • Copiez les serveurs de noms Cloudflare
    • Mettez-les à jour auprès du registraire de domaine (Hostinger, Namecheap, etc.)
    • Attendez la propagation (jusqu'à 24 heures)

Étape 3 : Créer un tunnel Cloudflare

Via le tableau de bord Cloudflare :

  1. Naviguez vers Zero Trust → Access → Tunnels
  2. Cliquez sur « Créer un tunnel »
  3. Sélectionnez Cloudflared
  4. Nom du tunnel : proxmox-tunnel
  5. Cliquez sur Enregistrer le tunnel

Copiez le jeton du tunnel - nécessaire pour l'installation de cloudflared.

Étape 4 : Installer cloudflared sur l'hôte Proxmox

# Download cloudflared
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb

# Install
sudo dpkg -i cloudflared-linux-amd64.deb

# Verify installation
cloudflared --version

Étape 5 : Configurer le tunnel

Créer un fichier de configuration :

# Create config directory
sudo mkdir -p /etc/cloudflared

# Create config file
sudo nano /etc/cloudflared/config.yml

Configuration (config.yml) :

tunnel: <TUNNEL_ID>
credentials-file: /etc/cloudflared/<TUNNEL_ID>.json

ingress:
  - hostname: proxmox.olyhome.site
    service: https://localhost:8006
    originRequest:
      noTLSVerify: true  # Requis pour le certificat auto-signé de Proxmox
  - service: http_status:404

Enregistrer les identifiants du tunnel :

# Create credentials file from Cloudflare dashboard
sudo nano /etc/cloudflared/<TUNNEL_ID>.json

# Paste JSON credentials from Cloudflare
# Set proper permissions
sudo chmod 600 /etc/cloudflared/<TUNNEL_ID>.json

Étape 6 : Configurer le nom d'hôte public dans Cloudflare

  1. Dans le tableau de bord Cloudflare, accédez à la configuration du tunnel
  2. Cliquez sur l'onglet Noms d'hôtes publics
  3. Cliquez sur Ajouter un nom d'hôte public :
    • Sous-domaine : proxmox
    • Domaine : olyhome.site
    • Service : https://localhost:8006
  4. Cliquez sur Enregistrer

Étape 7 : Démarrer le tunnel Cloudflare

# Test configuration
sudo cloudflared tunnel --config /etc/cloudflared/config.yml run

# Si la configuration réussit, installer comme service
sudo cloudflared service install

# Activer le démarrage automatique
sudo systemctl enable cloudflared

# Démarrer le service
sudo systemctl start cloudflared

# Vérifier le statut
sudo systemctl status cloudflared

Étape 8 : Configurer le SSL pour Proxmox (Facultatif)

Pour un SSL approprié sans avertissement, obtenez un certificat Let’s Encrypt :

  1. Dans l'interface utilisateur Proxmox :
    • Naviguez vers Datacenter → ACME
    • Ajouter un compte : Let’s Encrypt
    • Ajouter un plugin de défi DNS (Cloudflare)
  2. Ajouter un jeton d'API Cloudflare :
    • Le jeton nécessite la permission Zone:DNS:Edit
    • Ajouter à la configuration ACME de Proxmox
  3. Demander un certificat :
    • Nœud → Système → Certificats
    • Ajouter : ACME
    • Domaine : proxmox.olyhome.site
    • Défi : DNS
    • Commander le certificat

Problèmes rencontrés et solutions

Problème 1 : Le tunnel affiche l'état « DOWN » (Inactif)

Problème : Le tableau de bord Cloudflare affiche l'état du tunnel comme INACTIF avec l'Erreur 1033.

Cause : Le tunnel n'est pas correctement connecté au réseau de périphérie de Cloudflare.

Solution :

# Vérifier le statut du service cloudflared
sudo systemctl status cloudflared

# Afficher les journaux
sudo journalctl -u cloudflared -f

# Redémarrer le service
sudo systemctl restart cloudflared

# Vérifier que l'ID du tunnel correspond à la configuration
cloudflared tunnel list

Problème 2 : Erreur 502 Bad Gateway

Problème : L'accès à https://proxmox.olyhome.site renvoie une erreur 502.

Causes : 1. Le service Proxmox n'est pas en cours d'exécution 2. URL de service incorrecte dans la configuration 3. Problèmes de vérification TLS

Solutions :

# 1. Vérifier que Proxmox est en cours d'exécution
systemctl status pveproxy

# 2. Tester l'accès local
curl -k https://localhost:8006

# 3. Mettre à jour config.yml avec noTLSVerify
originRequest:
  noTLSVerify: true

# 4. Redémarrer cloudflared
sudo systemctl restart cloudflared

Problème 3 : Perte de connectivité réseau après le redémarrage

Problème : Proxmox perd la connectivité réseau après le redémarrage du système, le ping échoue.

Cause : La passerelle n'est pas appliquée correctement à partir de /etc/network/interfaces.

Solution :

# Ajouter manuellement la route par défaut
ip route add default via 192.168.11.1 dev vmbr0

# Vérifier la connectivité
ping 8.8.8.8

# Pour rendre permanent, assurez-vous que /etc/network/interfaces contienne :
auto vmbr0
iface vmbr0 inet static
    address 192.168.11.254/24
    gateway 192.168.11.1
    bridge-ports enp3s0
    bridge-stp off
    bridge-fd 0

Problème 4 : cloudflared ne démarre pas automatiquement

Problème : Le tunnel fonctionne lorsqu'il est exécuté manuellement mais ne démarre pas au démarrage.

Cause : Le service a été lancé avec --token au lieu de --config.

Solution :

# Désinstaller le service existant
sudo cloudflared service uninstall

# Réinstaller avec la configuration appropriée
sudo cloudflared service install

# Assurez-vous que le service systemd utilise le fichier de configuration
sudo nano /etc/systemd/system/cloudflared.service

# Vérifier la ligne ExecStart :
ExecStart=/usr/bin/cloudflared --config /etc/cloudflared/config.yml tunnel run

# Recharger systemd et redémarrer
sudo systemctl daemon-reload
sudo systemctl enable cloudflared
sudo systemctl restart cloudflared

Problème 5 : Le mode strict TLS bloque l'accès

Problème : Le mode SSL/TLS de Cloudflare défini sur « Full (Strict) » (Complet (Strict)) bloque l'accès en raison du certificat auto-signé.

Solution :

Option A : Changer le mode SSL Cloudflare (Correction rapide) : 1. Tableau de bord Cloudflare → SSL/TLS 2. Changer le mode de Full (Strict) à Full 3. Cela autorise les certificats auto-signés

Option B : Installer un certificat valide (Correction appropriée) : 1. Utiliser Proxmox ACME avec le défi DNS Cloudflare 2. Obtenir un certificat Let’s Encrypt 3. Garder le mode Full (Strict) pour une meilleure sécurité

cloudflared tunnel cleanup <TUNNEL_ID>

  1. Zero Trust → Access → Tunnels
  2. Vérifier que le tunnel affiche l'état « Sain »
  3. Consulter les Mesures pour les statistiques de trafic

Considérations de sécurité

1. Activer les politiques d'accès Cloudflare

Ajouter une couche d'authentification :

  1. Zero Trust → Access → Applications
  2. Créer une application : proxmox.olyhome.site
  3. Ajouter une politique :
    • Règle : E-mail
    • Valeur : your-email@domain.com
  4. Nécessite maintenant une connexion avant d'accéder à Proxmox

2. Restreindre l'accès par emplacement

Règles de la politique :
├── Inclure: Le pays est le Maroc (ou votre pays)
├── Exclure: Plages d'IP VPN connues
└── Exiger: Vérification par e-mail

3. Activer les journaux d'audit

Suivre toutes les tentatives d'accès : - Zero Trust → Logs → Access - Examiner les tentatives de connexion - Surveiller les activités suspectes

4. Utiliser une authentification Proxmox forte

# Activer la 2FA dans Proxmox
# Datacenter → Permissions → Utilisateurs
# Modifier l'utilisateur → Ajouter l'ATF

Optimisation des performances

Réduire la latence

# Dans config.yml, ajoutez :
originRequest:
  noTLSVerify: true
  connectTimeout: 10s
  httpHostHeader: proxmox.olyhome.site

Activer HTTP/2

Cloudflare utilise automatiquement HTTP/2 entre le client et le périmètre, améliorant les performances.

Surveillance et maintenance

Vérifications de santé régulières

# Créer un script de surveillance
#!/bin/bash
# check-tunnel.sh

if systemctl is-active --quiet cloudflared; then
    echo "✅ Le tunnel est en cours d'exécution"
else
    echo "❌ Le tunnel est inactif, redémarrage..."
    systemctl restart cloudflared
fi

# Ajouter à crontab
crontab -e
# */5 * * * * /root/check-tunnel.sh

Sauvegarde de la configuration

# Sauvegarder la configuration du tunnel
sudo tar czf cloudflared-backup-$(date +%Y%m%d).tar.gz \
    /etc/cloudflared/

# Stocker hors site (Dropbox, Google Drive, etc.)

Conclusion

Cloudflare Tunnel offre un accès à distance sécurisé et sans configuration à Proxmox sans exposer de ports. La configuration nécessite un paramétrage initial, mais aboutit à une solution robuste et prête pour la production.

Résultats finaux : - ✅ Accès sécurisé à https://proxmox.olyhome.site - ✅ Aucun port exposé sur le pare-feu - ✅ Certificats SSL gratuits - ✅ Redémarrage automatique du tunnel au démarrage - ✅ Protection DDoS incluse
