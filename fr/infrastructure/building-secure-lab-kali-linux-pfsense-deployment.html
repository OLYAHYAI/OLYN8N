Construction d'un laboratoire sécurisé : Déploiement de Kali Linux et pfSense

Aperçu du projet

La création d'un laboratoire de sécurité et de tests d'intrusion dédié est essentielle pour tout professionnel de la sécurité en devenir. Cet article documente le déploiement complet d'un environnement homelab sécurisé comprenant Kali Linux et pfSense, exécutés sur Proxmox VE.

Architecture du laboratoire

Proxmox Host
├── pfSense VM (Firewall/Router)
│   ├── WAN (vmbr0) → 192.168.11.253
│   └── LAN (vmbr1) → 192.168.1.1
├── Kali Linux VM (Penetration Testing)
│   └── Network: vmbr1 → 192.168.1.10
└── Ubuntu Server VM (Services)
    └── Network: vmbr1 → 192.168.1.20

Installation de la VM Kali Linux

Spécifications de la VM

Configuration Matérielle : - BIOS : SeaBIOS (après des problèmes de compatibilité OVMF) - Disque : 80 Go (contrôleur unique VirtIO SCSI) - CPU : 4 cœurs - RAM : 8 Go - Réseau : VirtIO (connecté à vmbr1) - Ordre de démarrage : Disque d'abord, CD-ROM ensuite

Pourquoi ces spécifications ?

Taille du Disque (80 Go) : - Installation de base de Kali : ~20 Go - Base de données d'outils et d'exploits : ~15 Go - Espace de travail et fichiers de projet : ~20 Go - Snapshots de VM et mises à jour : ~25 Go

CPU (4 cœurs) : - Balayage et exploitation parallèles - Cassage de mot de passe efficace - Performances de l'interface graphique fluides - Support des outils multithreadés

RAM (8 Go) : - Fonctionnement confortable de l'interface graphique - Plusieurs VMs pour les scénarios de laboratoire - Développement d'exploits en mémoire - Outils basés sur navigateur (Burp Suite)

Création de la VM dans Proxmox

Étape 1 : Télécharger l'ISO de Kali Linux 

# On Proxmox host
cd /var/lib/vz/template/iso

# Download latest Kali Linux
wget https://cdimage.kali.org/kali-2024.4/kali-linux-2024.4-installer-amd64.iso

# Verify checksum
wget https://cdimage.kali.org/kali-2024.4/SHA256SUMS
sha256sum -c SHA256SUMS 2>&1 | grep kali-linux-2024.4-installer-amd64.iso

Étape 2 : Créer la VM via l'interface utilisateur Proxmox

  1. Général:
    • VM ID: 101 (ou votre préférence)
    • Name: kali-pentest
    • Démarrer au démarrage : Non (démarrage manuel pour le travail en laboratoire)
  2. SE (Système d'Exploitation):
    • ISO Image: kali-linux-2024.4-installer-amd64.iso
    • Guest OS Type: Linux
    • Version: 6.x - 2.6 Kernel
  3. Système:
    • BIOS: SeaBIOS
    • Machine: i440fx (ou q35 pour un meilleur support matériel)
    • SCSI Controller: VirtIO SCSI Single
    • Agent Qemu : ✓ (activer après l'installation du SE)
  4. Disques:
    • Bus/Device: SCSI
    • Storage: local-lvm
    • Disk size: 80 GB
    • Cache : Write back (pour de meilleures performances)
    • Discard : ✓ (pour le thin provisioning)
    • Émulation SSD : ✓ (si l'hôte utilise un SSD)
  5. CPU:
    • Sockets: 1
    • Cores: 4
    • Type : host (ou kvm64 pour la compatibilité)
  6. Mémoire:
    • Memory: 8192 MB
    • Minimum : 4096 Mo (ballooning)
  7. Réseau:
    • Bridge: vmbr1 (réseau de laboratoire isolé)
    • Model: VirtIO (paravirtualized)
    • Pare-feu : Désactivé (pfSense gérera cela)

Étape 3 : Installer Kali Linux

  1. Start VM and open console
  2. Démarrer depuis l'installateur
  3. Sélectionner Installation Graphique
  4. Langue, Localisation, Clavier : Vos préférences
  5. Configuration Réseau:
    • Configurer automatiquement via DHCP (pfSense le fournira)
    • Hostname: kali-pentest
    • Domain: lab.local
  6. Configuration Utilisateur:
    • Nom complet : Testeur d'intrusion
    • Username: pentest
    • Mot de passe : Mot de passe fort requis
  7. Partitionnement:
    • Méthode : Guidée - utiliser le disque entier
    • Schéma : Tous les fichiers dans une seule partition
    • Écrire les changements : Oui
  8. Sélection de Logiciels:
    • ✓ Bureau Kali par défaut (XFCE recommandé pour la performance)
    • ✓ Top 10 des outils de sécurité
    • ✓ Grande collection d'outils (si l'espace disque le permet)
  9. Chargeur de Démarrage GRUB:
    • Installer sur : /dev/sda
    • Continue
  10. Redémarrer et se connecter

# Full system update

Installer l'Agent Invité Proxmox

# Install qemu-guest-agent
sudo apt install qemu-guest-agent -y

# Enable and start service
sudo systemctl enable qemu-guest-agent
sudo systemctl start qemu-guest-agent

# In Proxmox UI: VM → Options → QEMU Guest Agent → Enable

Installation de la VM pfSense

Spécifications de la VM

Configuration Matérielle : - BIOS : SeaBIOS - Disque : 20 Go (suffisant pour pfSense + paquets) - CPU : 2 cœurs (suffisant pour un petit laboratoire) - RAM : 4 Go (2 Go minimum, 4 Go confortable) - Interfaces Réseau : - vtnet0 (WAN) → vmbr0 - vtnet1 (LAN) → vmbr1

Création du Pont Virtuel (vmbr1)

Avant de créer la VM pfSense, assurez-vous que vmbr1 existe :

# On Proxmox host, edit network config
nano /etc/network/interfaces

# Add vmbr1 configuration:
auto vmbr1
iface vmbr1 inet manual
    bridge-ports none
    bridge-stp off
    bridge-fd 0
    bridge-vlan-aware yes
    bridge-vids 2-4094
    # No IP address - this is for VMs only

# Apply changes
ifreload -a

# Verify
ip link show vmbr1

Création de la VM pfSense

Télécharger l'ISO de pfSense

# On Proxmox host
cd /var/lib/vz/template/iso

# Download pfSense (community edition)
wget https://nyifiles.netgate.com/mirror/downloads/pfSense-CE-2.7.2-RELEASE-amd64.iso.gz

# Extract
gunzip pfSense-CE-2.7.2-RELEASE-amd64.iso.gz

Créer la VM via l'interface utilisateur Proxmox

  1. Général:
    • VM ID: 100
    • Name: pfsense-firewall
    • Démarrer au démarrage : Oui (infrastructure critique)
  2. SE (Système d'Exploitation):
    • ISO Image: pfSense-CE-2.7.2-RELEASE-amd64.iso
    • Type de SE Invité : Autre
  3. Système:
    • BIOS: SeaBIOS
    • SCSI Controller: VirtIO SCSI
  4. Disques:
    • Bus/Device: SCSI
    • Disk size: 20 GB
    • Cache : Write back
  5. CPU:
    • Cores: 2
  6. Mémoire:
    • Memory: 4096 MB
  7. Réseau:
    • Première Interface (WAN):
      • Bridge: vmbr0
      • Model: VirtIO
    • Après la création, ajouter la seconde interface (LAN):
      • Hardware → Add → Network Device
      • Bridge: vmbr1
      • Model: VirtIO

Installer pfSense

  1. Start VM and open console
  2. Accepter l'avis de droit d'auteur
  3. Installer pfSense : Sélectionner Install
  4. Keymap: Auto (ZFS)
  5. Partitionnement : Stripe (disque unique)
  6. Select disk: da0 (20GB)
  7. Confirmer : Yes
  8. L'installation est terminée
  9. Redémarrer

Configuration initiale de pfSense

Après le redémarrage, configurez les interfaces via la console :

Should VLANs be set up now? n

Enter the WAN interface name: vtnet0
Enter the LAN interface name: vtnet1

Proceed? y

Définir l'adresse IP du LAN :

Set interface(s) IP address
Select interface: 2 (LAN)

Configure IPv4 address via DHCP? n
Enter new LAN IPv4 address: 192.168.1.1
Enter new LAN IPv4 subnet: 24

Configure IPv6? n

Enable DHCP server on LAN? y
Enter start IP: 192.168.1.10
Enter end IP: 192.168.1.250

Accéder à l'interface Web de pfSense

Depuis la VM Kali ou une autre VM sur vmbr1 :

# Get DHCP lease from pfSense
sudo dhclient

# Verify IP
ip addr show

# Access pfSense GUI
https://192.168.1.1

# Default credentials:
# Username: admin
# Password: pfsense

Configuration du Réseau du Laboratoire

Schéma Réseau

Internet
   ↓
Router (192.168.11.1)
   ↓
Proxmox vmbr0 (192.168.11.254)
   ↓
pfSense WAN (192.168.11.253)
   ↓
pfSense LAN (192.168.1.1)
   ↓
vmbr1 (Réseau Interne du Laboratoire)
   ├── Kali Linux (192.168.1.10)
   ├── Ubuntu Server (192.168.1.20)
   └── Target VMs (192.168.1.50-100)

Configurer le Réseau Kali

# Edit network config
sudo nano /etc/network/interfaces

# Add static IP (optional, DHCP works too)
auto eth0
iface eth0 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 192.168.1.1

# Restart networking
sudo systemctl restart networking

# Or use network manager
sudo nmtui

Prise de Snapshots

Pourquoi un Snapshot ?

  • Expérimentation sécurisée : Revenir en arrière après les tests
  • Sauvegardes de configuration : Avant les changements majeurs
  • Récupération rapide : Restaurer après un échec

Créer un Snapshot via l'interface utilisateur Proxmox

  1. Select VM (Kali or pfSense)
  2. Aller à l'onglet Snapshots
  3. Cliquer sur Take Snapshot (Prendre un Snapshot)
  4. Name: post-install-clean ou working-config
  5. Description : « Installation propre, entièrement mise à jour »
  6. Inclure la RAM : Non (snapshot plus petit)

Créer un Snapshot via CLI

# Snapshot Kali VM (ID 101)
qm snapshot 101 post-install --description "Clean Kali install after updates"

# Snapshot pfSense VM (ID 100)
qm snapshot 100 baseline-config --description "Working pfSense configuration"

# List snapshots
qm listsnapshot 101

# Rollback to snapshot
qm rollback 101 post-install

Test du Laboratoire

Vérifier la Connectivité Réseau

# From Kali VM:

# Test LAN gateway
ping 192.168.1.1

# Test WAN gateway (through pfSense)
ping 8.8.8.8

# Test DNS resolution
nslookup google.com

Tests d'Intrusion de Base

# Scan internal network
nmap -sn 192.168.1.0/24

# Scan pfSense
nmap -sV -p 1-1000 192.168.1.1

# Test vulnerability scanning
nikto -h https://192.168.1.1

Conclusion

Ce déploiement de laboratoire sécurisé offre :

  • ✅ Un environnement de test d'intrusion isolé
  • ✅ Une segmentation réseau appropriée avec pfSense
  • ✅ Une station de travail Kali Linux entièrement fonctionnelle
  • ✅ Un filet de sécurité basé sur les snapshots
  • ✅ Une architecture évolutive pour des VM supplémentaires

Ce laboratoire sert de base pour : - Le test d'outils de sécurité - Le développement d'exploits - Les tests d'intrusion réseau - La préparation aux certifications (OSCP, CEH, etc.)

Prochaines étapes : Déployer des VM vulnérables (Metasploitable, DVWA) et configurer pfSense avec un IDS/IPS (Suricata/Snort).
