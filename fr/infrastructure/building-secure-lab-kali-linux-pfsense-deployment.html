Construction d'un laboratoire sécurisé : Déploiement de Kali Linux et pfSense

Aperçu du Projet

La création d'un laboratoire dédié aux tests d'intrusion et à la sécurité est essentielle pour tout professionnel de la sécurité aspirant. Cet article documente le déploiement complet d'un environnement homelab sécurisé comprenant Kali Linux et pfSense fonctionnant sur Proxmox VE.

Architecture du Laboratoire

Proxmox Host
├── pfSense VM (Firewall/Router)
│   ├── WAN (vmbr0) → 192.168.11.253
│   └── LAN (vmbr1) → 192.168.1.1
├── Kali Linux VM (Penetration Testing)
│   └── Network: vmbr1 → 192.168.1.10
└── Ubuntu Server VM (Services)
    └── Network: vmbr1 → 192.168.1.20

Installation de la VM Kali Linux

Spécifications de la VM

Configuration Matérielle : - BIOS: SeaBIOS (après des problèmes de compatibilité OVMF) - Disk: 80 Go (Contrôleur unique VirtIO SCSI) - CPU: 4 cœurs - RAM: 8 Go - Network: VirtIO (connecté à vmbr1) - Boot Order: Disque en premier, CD-ROM en second

Pourquoi ces spécifications ?

Taille du Disque (80 Go): - Installation de base Kali : ~20 Go - Base de données d'outils et d'exploits : ~15 Go - Espace de travail et fichiers de projet : ~20 Go - Snapshots de VM et mises à jour : ~25 Go

CPU (4 cœurs): - Scan et exploitation parallèles - Cassage de mot de passe efficace - Performance fluide de l'interface graphique - Support des outils multithreadés

RAM (8 Go): - Fonctionnement confortable de l'interface graphique - Multiples VMs pour les scénarios de laboratoire - Développement d'exploits en mémoire - Outils basés sur navigateur (Burp Suite)

Création de la VM dans Proxmox

Étape 1 : Téléchargement de l'ISO Kali Linux

# On Proxmox host
cd /var/lib/vz/template/iso

# Download latest Kali Linux
wget https://cdimage.kali.org/kali-2024.4/kali-linux-2024.4-installer-amd64.iso

# Verify checksum
wget https://cdimage.kali.org/kali-2024.4/SHA256SUMS
sha256sum -c SHA256SUMS 2>&1 | grep kali-linux-2024.4-installer-amd64.iso

Étape 2 : Création de la VM via l'interface utilisateur Proxmox

  1. Général:
    • VM ID: 101 (ou votre préférence)
    • Name: kali-pentest
    • Démarrer au boot : Non (démarrage manuel pour les travaux de laboratoire)
  2. OS:
    • ISO Image: kali-linux-2024.4-installer-amd64.iso
    • Type d'OS Invité : Linux
    • Version: 6.x - 2.6 Kernel
  3. Système:
    • BIOS: SeaBIOS
    • Machine: i440fx (ou q35 pour un meilleur support matériel)
    • SCSI Controller: VirtIO SCSI Single
    • Agent Qemu : ✓ (activer après l'installation de l'OS)
  4. Disques:
    • Bus/Device: SCSI
    • Storage: local-lvm
    • Disk size: 80 Go
    • Cache: Write back (pour de meilleures performances)
    • Discard: ✓ (pour le thin provisioning)
    • SSD emulation: ✓ (si l'hôte utilise un SSD)
  5. CPU:
    • Sockets: 1
    • Cores: 4
    • Type: host (ou kvm64 pour la compatibilité)
  6. Mémoire:
    • Memory: 8192 MB
    • Minimum: 4096 MB (ballooning)
  7. Réseau:
    • Bridge: vmbr1 (réseau de laboratoire isolé)
    • Model: VirtIO (paravirtualisé)
    • Firewall: Désactivé (pfSense gérera cela)

Étape 3 : Installation de Kali Linux

  1. Démarrer la VM et ouvrir la console
  2. Démarrer à partir de l'installateur
  3. Sélectionner Installation Graphique
  4. Langue, Emplacement, Clavier : Vos préférences
  5. Configuration Réseau:
    • Configurer automatiquement via DHCP (fourni par pfSense)
    • Hostname: kali-pentest
    • Domain: lab.local
  6. Configuration Utilisateur:
    • Nom complet: Penetration Tester
    • Username: pentest
    • Password: Mot de passe fort requis
  7. Partitionnement:
    • Méthode : Guidé - utiliser le disque entier
    • Schéma : Tous les fichiers dans une seule partition
    • Écrire les changements : Oui
  8. Sélection de Logiciels:
    • ✓ Bureau Kali par défaut (XFCE recommandé pour la performance)
    • ✓ Top 10 des outils de sécurité
    • ✓ Grande collection d'outils (si l'espace disque le permet)
  9. Chargeur de Démarrage GRUB:
    • Installer sur : /dev/sda
    • Continuer
  10. Redémarrer et se connecter

# Enable and start service

Installation de la VM pfSense

Spécifications de la VM

Configuration Matérielle : - BIOS: SeaBIOS - Disk: 20 Go (suffisant pour pfSense + paquets) - CPU: 2 cœurs (adéquat pour un petit labo) - RAM: 4 Go (2 Go minimum, 4 Go confortable) - Interfaces Réseau: - vtnet0 (WAN) → vmbr0 - vtnet1 (LAN) → vmbr1

Création du Pont Virtuel (vmbr1)

Avant de créer la VM pfSense, assurez-vous que vmbr1 existe :

# On Proxmox host, edit network config
nano /etc/network/interfaces

# Add vmbr1 configuration:
auto vmbr1
iface vmbr1 inet manual
    bridge-ports none
    bridge-stp off
    bridge-fd 0
    bridge-vlan-aware yes
    bridge-vids 2-4094
    # Pas d'adresse IP - ceci est uniquement pour les VMs

# Apply changes
ifreload -a

# Verify
ip link show vmbr1

Création de la VM pfSense

Téléchargement de l'ISO pfSense

# On Proxmox host
cd /var/lib/vz/template/iso

# Download pfSense (community edition)
wget https://nyifiles.netgate.com/mirror/downloads/pfSense-CE-2.7.2-RELEASE-amd64.iso.gz

# Extract
gunzip pfSense-CE-2.7.2-RELEASE-amd64.iso.gz

Créer la VM via l'interface utilisateur Proxmox

  1. Général:
    • VM ID: 100
    • Name: pfsense-firewall
    • Démarrer au boot : Oui (infrastructure critique)
  2. OS:
    • ISO Image: pfSense-CE-2.7.2-RELEASE-amd64.iso
    • Type d'OS Invité : Other
  3. Système:
    • BIOS: SeaBIOS
    • SCSI Controller: VirtIO SCSI
  4. Disques:
    • Bus/Device: SCSI
    • Disk size: 20 Go
    • Cache: Write back
  5. CPU:
    • Cores: 2
  6. Mémoire:
    • Memory: 4096 MB
  7. Réseau:
    • Première Interface (WAN):
      • Bridge: vmbr0
      • Model: VirtIO
    • Après la création, ajouter une seconde interface (LAN):
      • Hardware → Add → Network Device
      • Bridge: vmbr1
      • Model: VirtIO

Installation de pfSense

  1. Démarrer la VM et ouvrir la console
  2. Accepter l'avis de droit d'auteur
  3. Installer pfSense: Sélectionner Install
  4. Keymap: Auto (ZFS)
  5. Partitionnement: Stripe (disque unique)
  6. Sélectionner le disque : da0 (20GB)
  7. Confirmer: Yes
  8. L'installation est terminée
  9. Redémarrer

Configuration Initiale de pfSense

Après le redémarrage, configurer les interfaces via la console :

Les VLANs doivent-ils être configurés maintenant ? n

Entrez le nom de l'interface WAN : vtnet0
Entrez le nom de l'interface LAN : vtnet1

Continuer ? y

Définir l'adresse IP LAN :

Définir l'adresse IP de l'interface/des interfaces
Sélectionner l'interface : 2 (LAN)

Configurer l'adresse IPv4 via DHCP ? n
Entrez la nouvelle adresse IPv4 LAN : 192.168.1.1
Entrez le nouveau sous-réseau IPv4 LAN : 24

Configurer IPv6 ? n

Activer le serveur DHCP sur le LAN ? y
Entrez l'IP de début : 192.168.1.10
Entrez l'IP de fin : 192.168.1.250

Configurer le Réseau Kali

# Modifier la configuration réseau
sudo nano /etc/network/interfaces

# Ajouter une IP statique (optionnel, le DHCP fonctionne aussi)
auto eth0
iface eth0 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 192.168.1.1

# Redémarrer le réseau
sudo systemctl restart networking

# Ou utiliser le gestionnaire de réseau
sudo nmtui

Prise de Snapshots

Pourquoi prendre un Snapshot ?

  • Expérimentation sécurisée: Revenir après les tests
  • Sauvegardes de configuration: Avant des changements majeurs
  • Récupération rapide: Retour arrière en cas d'échec

Créer un Snapshot via l'interface utilisateur Proxmox

  1. Sélectionner la VM (Kali ou pfSense)
  2. Aller à l'onglet Snapshots
  3. Cliquer sur Take Snapshot
  4. Nom: post-install-clean ou working-config
  5. Description: “Installation fraîche, entièrement mise à jour”
  6. Include RAM: Non (snapshot plus petit)

Créer un Snapshot via la CLI

# Snapshot de la VM Kali (ID 101)
qm snapshot 101 post-install --description "Clean Kali install after updates"

# Snapshot de la VM pfSense (ID 100)
qm snapshot 100 baseline-config --description "Working pfSense configuration"

# Lister les snapshots
qm listsnapshot 101

# Revenir au snapshot
qm rollback 101 post-install

# Depuis la VM Kali :

# Tester la passerelle WAN (via pfSense)

Tests d'Intrusion Basiques

# Scanner le réseau interne
nmap -sn 192.168.1.0/24

# Scanner pfSense
nmap -sV -p 1-1000 192.168.1.1

# Tester le scan de vulnérabilités
nikto -h https://192.168.1.1

Conclusion

Ce déploiement de laboratoire sécurisé offre :

  • ✅ Environnement de test d'intrusion isolé
  • ✅ Segmentation réseau appropriée avec pfSense
  • ✅ Poste de travail Kali Linux entièrement fonctionnel
  • ✅ Filet de sécurité basé sur les snapshots
  • ✅ Architecture évolutive pour des VMs additionnelles

Ce laboratoire sert de fondation pour : - Le test d'outils de sécurité - Le développement d'exploits - Les tests d'intrusion réseau - La préparation aux certifications (OSCP, CEH, etc.)
