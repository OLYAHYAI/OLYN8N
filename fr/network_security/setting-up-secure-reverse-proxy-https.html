Configuration d'un Proxy Inverse Sécurisé HTTPS

2025-11-19
5 min de lecture
Sécurité Réseau
Reverse Proxy, Homelab, Nginx, HTTPS, Docker, pfSense, Let's Encrypt

Le Gardien Numérique de Mon Homelab : Configuration d'un Proxy Inverse Sécurisé avec HTTPS

Objectifs

  • Accéder à tous vos services web internes (Proxmox, n8n, et bases de données) de manière sécurisée via des domaines HTTPS en utilisant un proxy inverse.
  • Utiliser Let’s Encrypt pour des certificats SSL gratuits et automatisés.

Pourquoi Utiliser un Proxy Inverse ?

Un proxy inverse agit comme une passerelle centralisée vers vos services homelab, offrant plusieurs avantages critiques :

  • Gestion SSL Unifiée: Point unique pour la gestion des certificats
  • Couche de Sécurité: Protection contre l'exposition directe des services
  • Domaines Personnalisés: URL claires au lieu de combinaisons IP:PORT
  • Équilibrage de Charge: Distribution du trafic sur plusieurs serveurs backend

Aperçu de l'Architecture

Internet/LAN
    ↓
pfSense Firewall
    ↓
Nginx Proxy Manager (Reverse Proxy)
    ↓
├── Proxmox (192.168.x.x:8006)
├── n8n (192.168.x.x:5678)
├── Databases (192.168.x.x:XXXX)
└── Other Services

Étape 1 : Déploiement de la VM/Conteneur du Proxy Inverse

Option A : VM Légère

  • Créez une nouvelle VM sur Proxmox (Debian/Ubuntu recommandé)
  • Allouez des ressources minimales : 1 CPU, 2 Go de RAM, 10 Go de stockage
  • Installez Docker et Docker Compose

Option B : Conteneur LXC (Recommandé)

  • Utilisez des scripts d'aide Proxmox pour un déploiement plus rapide
  • Utilisation plus efficace des ressources
  • Gestion plus facile

Installation de Docker et Docker Compose

# Update system
sudo apt update && sudo apt upgrade -y

# Install Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Install Docker Compose
sudo apt install docker-compose -y

# Add user to docker group
sudo usermod -aG docker $USER

Étape 2 : Installation de Nginx Proxy Manager

Configuration Docker Compose

Créez docker-compose.yml:

version: '3'
services:
  app:
    image: 'jc21/nginx-proxy-manager:latest'
    container_name: nginx-proxy-manager
    restart: unless-stopped
    ports:
      - '80:80'      # HTTP
      - '81:81'      # Admin UI
      - '443:443'    # HTTPS
    environment:
      DB_MYSQL_HOST: "db"
      DB_MYSQL_PORT: 3306
      DB_MYSQL_USER: "npm"
      DB_MYSQL_PASSWORD: "npm"
      DB_MYSQL_NAME: "npm"
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
  db:
    image: 'jc21/mariadb-aria:latest'
    container_name: npm-database
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: 'npm'
      MYSQL_DATABASE: 'npm'
      MYSQL_USER: 'npm'
      MYSQL_PASSWORD: 'npm'
    volumes:
      - ./data/mysql:/var/lib/mysql

Lancement de Nginx Proxy Manager

docker-compose up -d

Accès à l'Interface d'Administration

  • Naviguez vers : http://<proxy-vm-ip>:81
  • Identifiants par défaut :
    • Email : admin@example.com
    • Mot de passe : changeme
  • Changez immédiatement ces identifiants après la première connexion !

Étape 3 : Configuration du DNS Interne

Utilisation du Résolveur DNS pfSense

  1. Naviguez vers Services > DNS Resolver
  2. Activez le Résolveur DNS s'il n'est pas déjà activé
  3. Sous Host Overrides, ajoutez des entrées :
    • Host: proxmox, Domain: lab, IP: <proxy-ip>
    • Host: n8n, Domain: lab, IP: <proxy-ip>
    • Host: db, Domain: lab, IP: <proxy-ip>

Alternative : Fichier Hosts Local

Pour les tests, modifiez votre fichier local /etc/hosts:

<proxy-ip>  proxmox.lab
<proxy-ip>  n8n.lab
<proxy-ip>  db.lab

Étape 4 : Ajout des Services comme Hôtes Proxy

Dans l'interface utilisateur de Nginx Proxy Manager

  1. Cliquez sur “Proxy Hosts” → “Add Proxy Host”
  2. Onglet Détails:
    • Domain Names: n8n.lab
    • Scheme: http
    • Forward Hostname/IP: <n8n-container-ip>
    • Forward Port: 5678
    • Block Common Exploits: ✓
    • Websockets Support: ✓ (important pour n8n)
  3. Répétez pour les autres services :
    • Proxmox: Port 8006, Scheme https
    • Databases: Port variable, Scheme http

Étape 5 : Demande de Certificats SSL

Activation de Let’s Encrypt

  1. Dans l'entrée Proxy Host, allez à l'Onglet SSL
  2. Sélectionnez “Request a new SSL Certificate” (Demander un nouveau certificat SSL)
  3. Options :
    • Force SSL: ✓
    • HTTP/2 Support: ✓
    • HSTS Enabled: ✓ (recommandé)
    • Email for Let’s Encrypt: Votre adresse e-mail
  4. Cliquez sur Save

Let’s Encrypt va automatiquement : - Générer le certificat SSL - Configurer HTTPS - Configurer le renouvellement automatique (tous les 90 jours)

Défi DNS (Pour les Certificats Wildcard)

Si vous possédez un domaine et souhaitez *.yourdomain.com :

  1. Utilisez la méthode du défi DNS
  2. Configurez les identifiants du fournisseur DNS dans NPM
  3. Demandez le certificat wildcard : *.yourdomain.com

Étape 6 : Ouverture du Port 443 dans le Pare-feu

Configuration pfSense

  1. Naviguez vers Firewall > NAT > Port Forward
  2. Créez la règle :
    • Interface: WAN
    • Protocol: TCP
    • Destination Port Range: 443
    • Redirect Target IP: <proxy-vm-ip>
    • Redirect Target Port: 443
    • Description: Reverse Proxy HTTPS
  3. Créez la règle de pare-feu correspondante si nécessaire

Pare-feu VM/Conteneur

Si vous utilisez UFW :

sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 81/tcp  # Admin UI (restrict to LAN only!)

Étape 7 : Tester l'Accès

Test Interne

Depuis votre LAN :

curl -k https://n8n.lab
curl -k https://proxmox.lab:8006

Test par Navigateur

  1. Ouvrez votre navigateur
  2. Naviguez vers https://n8n.lab
  3. Vérifiez :
    • ✅ Certificat SSL valide (cadenas vert)
    • ✅ Le service se charge correctement
    • ✅ Aucun avertissement de certificat

Durcissement de la Sécurité

Restreindre l'Accès à l'Interface d'Administration

  1. Dans Nginx Proxy Manager :

    • Access Lists → Créer une nouvelle liste
    • Authorization: Exiger un nom d'utilisateur/mot de passe
    • Appliquer à l'hôte proxy de l'interface d'administration

  2. Règles de pare-feu :

    # Allow admin UI only from LAN
sudo ufw deny 81/tcp
sudo ufw allow from 192.168.0.0/16 to any port 81

Activer Fail2Ban

Protégez-vous contre les attaques par force brute :

sudo apt install fail2ban -y

# Create NPM filter
sudo nano /etc/fail2ban/filter.d/npm.conf
[Definition]
failregex = ^<HOST> .* "(GET|POST|HEAD).*HTTP.*" (4|5)\d\d
ignoreregex =

Surveillance et Maintenance

Vérification de l'Expiration des Certificats

Nginx Proxy Manager renouvelle automatiquement les certificats 30 jours avant leur expiration.

Vérifiez dans l'interface utilisateur : - SSL Certificates → Vérifiez les dates d'expiration

Surveillance des Journaux

# View NPM logs
docker logs -f nginx-proxy-manager

# Access logs location
cd ./data/logs

Surveillance des Performances

Ajoutez une surveillance avec : - Prometheus + Grafana - Uptime Kuma pour la disponibilité des services

Problèmes Courants et Solutions

Problème 1 : Échec de la Demande de Certificat

Problème : Le défi Let’s Encrypt échoue

Solution : - Vérifiez que le port 80 est accessible depuis Internet - Vérifiez la résolution DNS - Essayez le défi HTTP au lieu du défi DNS

Problème 2 : Échec de la Connexion WebSocket

Problème : Les fonctionnalités en temps réel ne fonctionnent pas (n8n, console Proxmox)

Solution : - Activez Websockets Support dans l'hôte proxy - Ajoutez une configuration Nginx personnalisée : nginx proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade";

Problème 3 : Accès au WebUI Proxmox Refusé

Problème : Impossible d'accéder à Proxmox via le proxy

Solution : - Réglez le Forward Scheme sur https - Ajoutez des en-têtes personnalisés : nginx proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr;

Conclusion

La configuration d'un proxy inverse avec Nginx Proxy Manager offre un accès centralisé et sécurisé à tous vos services homelab. La combinaison de certificats SSL automatisés, de domaines personnalisés et d'une gestion unifiée crée une infrastructure professionnelle à la fois sécurisée et facile à maintenir.

Cette configuration sert de gardien numérique à votre homelab, protégeant les services tout en offrant un accès pratique. Avec une configuration appropriée et un durcissement de la sécurité, vous pouvez exposer en toute sécurité certains services à Internet tout en maintenant une protection robuste contre les accès non autorisés.

Rappel : La sécurité est un processus continu. Mettez régulièrement à jour votre proxy inverse, surveillez les journaux et examinez les modèles d'accès.
