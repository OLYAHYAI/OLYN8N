Homelab Session Notes: Proxmox VE Initial Setup and Network Configuration

Session Overview

Date: October 22, 2025 Focus: Proxmox VE installation, network configuration, and initial VM preparation Hardware: Lenovo ThinkCentre M920q

Hardware Specifications

Server Details

Model: Lenovo ThinkCentre M920q (Tiny Form Factor)

Specifications: - CPU: Intel i5-8500T (6 cores, 9MB cache, 35W TDP) - RAM: 16 GB DDR4 - Primary Storage: SSD 512 GB (NVMe or SATA) - Secondary Storage: HDD 250 GB (for backups/ISO storage) - Network: Gigabit Ethernet (enp3s0) - Form Factor: 1.0L ultra-compact

Why This Hardware: - Low power consumption (35W TDP ideal for 24/7 operation) - Compact form factor (perfect for homelab) - Sufficient RAM for multiple VMs - Dual storage for system/data separation - Excellent price/performance ratio on used market

Network Configuration

Proxmox Host IP: 192.168.11.254 Gateway/Router: 192.168.11.1 Subnet Mask: 255.255.255.0 (/24) Network: 192.168.11.0/24

Step 1: Accessing Proxmox Web Interface

Initial Access

# From web browser on local network
https://192.168.11.254:8006

# Login credentials
Username: root
Domain: pam (Linux PAM authentication)
Password: [set during installation]

First Login Checklist

Verify: - [ ] Web interface loads (ignore SSL warning for self-signed cert) - [ ] Can login with root credentials - [ ] Dashboard shows server resources correctly - [ ] Network interface shows correct IP

Post-Login Configuration

Disable Enterprise Repository (if using free version):

# SSH into Proxmox
ssh root@192.168.11.254

# Edit sources list
nano /etc/apt/sources.list.d/pve-enterprise.list

# Comment out enterprise repo
# deb https://enterprise.proxmox.com/debian/pve bullseye pve-enterprise

# Add no-subscription repository
echo "deb http://download.proxmox.com/debian/pve bullseye pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list

# Update
apt update

Step 2: Direct Ethernet Connection

Physical Setup

Topology:

Laptop (192.168.11.1)
    ↓ Ethernet Cable
Proxmox Server (192.168.11.254)

Manual IP Configuration

On Laptop (assuming Linux):

# Assign static IP to ethernet interface
sudo ip addr add 192.168.11.1/24 dev eth0
sudo ip link set eth0 up

# Test connectivity
ping 192.168.11.254

# Expected: replies from 192.168.11.254

Purpose: - Direct management connection - No dependency on router/DHCP - Useful for troubleshooting network issues - Simulates isolated lab network

Step 3: Wireless Network Verification

Check Available Interfaces

# List all network interfaces
ip a

# Expected output includes:
# lo: loopback
# enp3s0: ethernet (primary)
# wlp0s20f3: wireless (if present)

Detect Network Hardware

# List PCI network devices
lspci | grep -i network

# Expected output:
# 00:1f.6 Ethernet controller: Intel Corporation...
# (May show wireless if available)

# Check wireless interfaces
iw dev

# List USB devices (for USB WiFi adapters)
lsusb | grep -i wireless

WiFi Configuration (if applicable)

# Install wireless tools
apt install wpasupplicant wireless-tools -y

# Scan for networks
iwlist scan

# Configure wpa_supplicant
wpa_passphrase "SSID" "password" >> /etc/wpa_supplicant/wpa_supplicant.conf

# Connect
wpa_supplicant -B -i wlp0s20f3 -c /etc/wpa_supplicant/wpa_supplicant.conf
dhclient wlp0s20f3

Note: WiFi on Proxmox host generally not recommended for VM network bridging.

Step 4: Changing Proxmox IP Address

Edit Network Configuration

# Edit network interfaces file
nano /etc/network/interfaces

Configuration Example:

# Loopback interface
auto lo
iface lo inet loopback

# Primary network interface
auto enp3s0
iface enp3s0 inet manual

# Bridge for VMs (management network)
auto vmbr0
iface vmbr0 inet static
    address 192.168.11.254/24
    gateway 192.168.11.1
    bridge-ports enp3s0
    bridge-stp off
    bridge-fd 0
    bridge-vlan-aware yes
    # DNS servers
    dns-nameservers 8.8.8.8 8.8.4.4

Apply Network Changes

# Method 1: Restart networking service
systemctl restart networking

# Method 2: Reload interfaces (no disruption to VMs)
ifreload -a

# Method 3: Full reboot (safest)
reboot

Verify New Configuration

# Check IP address
ip addr show vmbr0

# Test gateway
ping 192.168.11.1

# Test internet
ping 8.8.8.8
ping google.com

Step 5: Uploading ISO Images

Method 1: Web Interface Upload

Via Proxmox UI:

  1. Navigate to: Datacenter → Node → local (storage) → Content
  2. Click Upload
  3. Select ISO image
  4. Choose file from local computer
  5. Click Upload

Pros: - User-friendly - No command-line needed - Visual progress bar

Cons: - Slow for large files - Requires stable network connection - Browser-dependent

Method 2: SCP (Secure Copy)

From laptop/workstation:

# Upload single ISO
scp /path/to/file.iso root@192.168.11.254:/var/lib/vz/template/iso/

# Upload multiple ISOs
scp *.iso root@192.168.11.254:/var/lib/vz/template/iso/

# With progress meter
rsync --progress -avz file.iso root@192.168.11.254:/var/lib/vz/template/iso/

Pros: - Faster for large files - Can resume interrupted transfers (with rsync) - Scriptable

Cons: - Requires SSH access - Command-line knowledge needed

Method 3: Direct Download on Proxmox

# SSH into Proxmox
ssh root@192.168.11.254

# Navigate to ISO directory
cd /var/lib/vz/template/iso/

# Download directly
wget https://releases.ubuntu.com/22.04/ubuntu-22.04.3-live-server-amd64.iso

# Or with aria2 (faster, multi-connection)
aria2c -x 16 https://releases.ubuntu.com/22.04/ubuntu-22.04.3-live-server-amd64.iso

Pros: - Fastest method - No transfer overhead - Direct from source

Cons: - Requires internet on Proxmox host - Uses Proxmox bandwidth

Step 6: LLM Model Selection

Recommended Models for 16GB RAM

Tier 1: Lightweight (2-4 GB RAM)

Mistral 7B Q4: - Size: ~4.3 GB - Performance: Excellent general purpose - Use case: Code, chat, general queries

Phi-2 (2.7B): - Size: ~1.6 GB - Performance: Fast, good for coding - Use case: Code completion, quick queries

Gemma 2B: - Size: ~1.4 GB - Performance: Very fast - Use case: Lightweight tasks, testing

Compatible Tools

Ollama:

# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull model
ollama pull mistral:7b

# Run model

llama.cpp:

Step 7: Docker + MCP + LLM Installation

Install Docker

# Update system
apt update && apt upgrade -y

# Install dependencies
apt install -y apt-transport-https ca-certificates curl gnupg lsb-release

# Add Docker GPG key
curl -fsSL https://download.docker.com/linux/debian/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Add Docker repository
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker
apt update
apt install -y docker-ce docker-ce-cli containerd.io

# Verify installation
docker --version
docker run hello-world

Deploy MCP

# Pull MCP image
docker pull ghcr.io/modularcopilot/mcp:latest

# Run MCP container
docker run -d \
  --name mcp \
  -p 5000:5000 \
  -v $(pwd)/mcp_data:/app/data \
  ghcr.io/modularcopilot/mcp:latest

# Check logs
docker logs -f mcp

Install Ollama for LLM Management

# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull models
ollama pull mistral:7b
ollama pull phi

# List installed models
ollama list

# Run model
ollama run mistral:7b

Notes and Best Practices

Network Configuration

  • Always use bridged networking (vmbr0) for VM management
  • Create separate bridge (vmbr1) for isolated VM networks
  • Document IP allocations in homelab notebook
  • Test gateway connectivity after IP changes

Storage Management

  • Use SSD for Proxmox system and VM disks
  • Use HDD for backups and ISO storage
  • Enable thin provisioning for efficient space usage
  • Regular snapshots before major changes

Security

  • Change default passwords immediately
  • Disable enterprise repo to avoid update errors
  • Keep Proxmox updated (apt update && apt upgrade)
  • Use SSH keys instead of passwords where possible
  • Consider 2FA for Proxmox web interface
