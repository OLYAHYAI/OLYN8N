HDD/SSD Health Monitoring and Analysis Guide

Tags: #storage #hardware #maintenance #smart

Quick Health Check Commands

# Basic disk information
lsblk
lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT

# Check SMART status (quick)
smartctl -H /dev/sda

# Full SMART analysis (detailed)
smartctl -a /dev/sda

# Check for disk errors in system log
dmesg | grep -i "error\|fail" | grep sda

# Monitor disk I/O in real-time
iostat -x 1

Understanding SMART Attributes

Critical Attributes to Monitor

Attribute ID What It Means Danger Level
Reallocated_Sector_Ct 5 Bad sectors remapped >100 = Warning
>500 = Critical
Current_Pending_Sector 197 Sectors waiting to be reallocated >0 = Warning
>10 = Critical
Offline_Uncorrectable 198 Sectors that can’t be read >0 = Warning
>10 = Critical
Reported_Uncorrect 187 Uncorrectable errors >0 = Warning
>100 = Critical
UDMA_CRC_Error_Count 199 Cable/connection errors >10 = Check cables
Spin_Retry_Count 10 Failed spin-up attempts >0 = Warning
Raw_Read_Error_Rate 1 Read errors before ECC Vendor specific

Health Status Interpretation

PASSED: - All attributes within thresholds - Doesn’t mean disk is healthy, just not failing yet

FAILING: - One or more attributes below threshold - REPLACE IMMEDIATELY

Step-by-Step Disk Health Analysis

Step 1: Check if Disk is Detected

# List all disks
lsblk

# Expected output shows disk name, size, type
# Example:
# sda     8:0    0  250G  0 disk

If disk not shown: - Check physical connection - Check BIOS/UEFI settings - Try different SATA port/cable

Step 2: Quick SMART Health Check

# Quick health check
smartctl -H /dev/sda

Possible Results:

✅ PASSED:

SMART overall-health self-assessment test result: PASSED
  • Disk passed basic health test
  • Continue to detailed analysis

❌ FAILING:

SMART overall-health self-assessment test result: FAILED
  • CRITICAL: Backup data immediately
  • Replace disk ASAP

⚠️ Unknown/Not Supported:

Device does not support SMART
  • Very old disk or USB enclosure blocking SMART
  • Use other diagnostic methods

Step 3: Detailed SMART Analysis

# Get full SMART report
smartctl -a /dev/sda > /tmp/smart_report_$(date +%Y%m%d).txt

# View the report
cat /tmp/smart_report_$(date +%Y%m%d).txt

Key Sections to Check:

A. Device Information

Model Family:     Seagate Momentus 5400.4
Device Model:     ST9250827AS
Serial Number:    5RG0J6QZ
User Capacity:    250,059,350,016 bytes [250 GB]
Rotation Rate:    5400 rpm
  • Note the model and age
  • Laptop HDDs (Momentus) wear faster than desktop HDDs
  • 5400 RPM is slow (laptop drive)

B. SMART Attributes Table

ID# ATTRIBUTE_NAME          VALUE WORST THRESH TYPE      WHEN_FAILED RAW_VALUE
  5 Reallocated_Sector_Ct   087   087   036    Pre-fail  -           534
197 Current_Pending_Sector  099   078   000    Old_age   -           33
198 Offline_Uncorrectable   099   078   000    Old_age   -           33
187 Reported_Uncorrect      001   001   000    Old_age   -           1582

How to Read: - VALUE: Current normalized value (100 = new, 0 = worst) - WORST: Lowest value ever recorded - THRESH: Threshold - if VALUE < THRESH, attribute failed - RAW_VALUE: Actual count (this is what matters!)

Red Flags: - Reallocated_Sector_Ct RAW > 100 - Current_Pending_Sector RAW > 0 - Reported_Uncorrect RAW > 100 - Any WHEN_FAILED showing “FAILING_NOW”

Step 4: Check System Logs for Disk Errors

# Check dmesg for disk errors
dmesg | grep -i sda | grep -i "error\|fail"

# Common error patterns:

Critical Errors:

# DID_BAD_TARGET - Disk not responding
sd 0:0:0:0: [sda] FAILED Result: hostbyte=DID_BAD_TARGET

→ Disk/controller failure, cable issue, or dead disk

# Medium Error - Unrecoverable read error
sd 0:0:0:0: [sda] Sense Key : Medium Error
Add. Sense: Unrecovered read error

→ Bad sectors, disk surface damage

# I/O error - Can't read/write
I/O error, dev sda, sector X op 0x0:(READ)
Buffer I/O error on dev sda

→ Physical media failure

Step 5: Run SMART Self-Tests

# Short test (1-2 minutes)
smartctl -t short /dev/sda

# Wait for completion
sleep 120

# Check results
smartctl -l selftest /dev/sda

# Long test (1-2 hours) - more thorough
smartctl -t long /dev/sda

# Check status
smartctl -a /dev/sda | grep -A 10 "Self-test execution status"

Test Results:

✅ Completed without error: - Disk passed all tests - Still monitor regularly

⚠️Completed: read element of test failed: - Found bad sectors during test - Consider replacement

❌ Aborted by host / interrupted: - Test couldn’t complete - Severe disk problems

Health Assessment Decision Matrix

When to Keep Using Disk

✅ All conditions must be true: - SMART status: PASSED - Reallocated sectors: 0-10 - Pending sectors: 0 - Uncorrectable errors: 0 - Error count: < 100 - No system log errors - Self-tests pass - Performance normal

When to Monitor Closely (Weekly)

⚠️ Any of these: - Reallocated sectors: 11-100 - Pending sectors: 1-5 - Uncorrectable errors: 1-10 - Error count: 100-500 - Occasional I/O errors - Performance degraded 10-20%

When to Replace Immediately

❌ Any of these: - SMART status: FAILED - Reallocated sectors: > 100 - Pending sectors: > 5 - Uncorrectable errors: > 10 - Error count: > 500 - Frequent DID_BAD_TARGET errors - Can’t mount filesystem - Performance degraded > 20% - Clicking/grinding noises

Automated Monitoring Script

#!/bin/bash
# Save as: /usr/local/bin/disk-health-check.sh
# Run daily via cron

LOG_FILE="/var/log/disk-health.log"
ALERT_EMAIL="root"

echo "=== Disk Health Check - $(date) ===" | tee -a $LOG_FILE

for DISK in /dev/sd? /dev/nvme?n?; do
    [ -e "$DISK" ] || continue
    
    echo "Checking $DISK..." | tee -a $LOG_FILE
    
    # Check SMART support
    if ! smartctl -i $DISK &>/dev/null; then
        echo "  SMART not supported on $DISK" | tee -a $LOG_FILE
        continue
    fi
    
    # Health check
    HEALTH=$(smartctl -H $DISK | grep "SMART overall-health" | awk '{print $NF}')
    echo "  Health: $HEALTH" | tee -a $LOG_FILE
    
    # Check critical attributes
    REALLOCATED=$(smartctl -A $DISK | grep "Reallocated_Sector" | awk '{print $10}')
    PENDING=$(smartctl -A $DISK | grep "Current_Pending" | awk '{print $10}')
    UNCORRECTABLE=$(smartctl -A $DISK | grep "Offline_Uncorrectable" | awk '{print $10}')
    
    echo "  Reallocated sectors: $REALLOCATED" | tee -a $LOG_FILE
    echo "  Pending sectors: $PENDING" | tee -a $LOG_FILE
    echo "  Uncorrectable sectors: $UNCORRECTABLE" | tee -a $LOG_FILE
    
    # Alert conditions
    if [ "$HEALTH" != "PASSED" ]; then
        echo "ALERT: $DISK health check FAILED!" | mail -s "CRITICAL: Disk Failure" $ALERT_EMAIL
    fi
    
    if [ "${REALLOCATED:-0}" -gt 100 ]; then
        echo "ALERT: $DISK has $REALLOCATED reallocated sectors!" | mail -s "WARNING: Disk Degradation" $ALERT_EMAIL
    fi
    
    if [ "${PENDING:-0}" -gt 0 ]; then
        echo "ALERT: $DISK has $PENDING pending sectors!" | mail -s "WARNING: Disk Errors" $ALERT_EMAIL
    fi
    
    echo "" | tee -a $LOG_FILE
done

echo "===================================" | tee -a $LOG_FILE

Installation:

# Make executable
chmod +x /usr/local/bin/disk-health-check.sh

# Add to daily cron
echo "0 8 * * * /usr/local/bin/disk-health-check.sh" | crontab -

Monitoring Schedule

Daily (Automated)

# Add to cron: /etc/cron.daily/disk-check
#!/bin/bash
for disk in /dev/sd? /dev/nvme?n?; do
    [ -e "$disk" ] || continue
    smartctl -H $disk | grep -q PASSED || \
        echo "WARNING: $disk health check failed!" | mail -s "Disk Alert" root
done

Weekly (Manual)

# Check SMART status
smartctl -a /dev/sda | grep -E "Reallocated|Pending|Uncorrect|Error"

# Check system logs
dmesg | grep -i "sda.*error" | tail -20

# Check performance
iostat -x 1 5

Monthly (Detailed)

# Run short self-test
smartctl -t short /dev/sda
sleep 120
smartctl -l selftest /dev/sda

# Full SMART report
smartctl -a /dev/sda > /var/log/smart_$(date +%Y%m%d).log

# Compare to previous month
diff /var/log/smart_$(date -d "1 month ago" +%Y%m%d).log \
     /var/log/smart_$(date +%Y%m%d).log

Disk Lifespan Expectations

Type Average Lifespan Factors
Consumer HDD 3-5 years Usage, temp, quality
Enterprise HDD 5-7 years Built for 24/7 use
Consumer SSD 5-10 years Write cycles, TBW
Enterprise SSD 5-10+ years Higher endurance

Warning Signs of Age: - 5+ years old → Monitor closely - 7+ years old → Plan replacement - 10+ years old → Replace proactively

Key Takeaways

  1. Monitor regularly - SMART data predicts 70% of failures
  2. Act on warnings - Don’t wait for complete failure
  3. Backups are critical - RAID is not a backup
  4. Replace proactively - Old disks will fail
  5. Test your backups - Verify you can restore
  6. Document everything - Keep maintenance logs
